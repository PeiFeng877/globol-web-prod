# Globol 核心监控方案（精简版）

面向当前代码与页面结构，**只保留最核心指标**，聚焦“SEO 拉新 + 下载转化”。

## 1. 为什么要监控

- **验证 SEO 是否有效**：内容页带来的是访问，还是能推动下载。
- **聚焦唯一转化目标**：当前网站的核心目标是下载，其他指标只服务于此。
- **避免分散注意力**：只跟踪能驱动决策的少数指标。

## 2. 如何监控

### 2.1 采集方式

- 使用 **Firebase Analytics（GA4）**。
- 仅做最小化埋点：
  - `page_view`（必须手动，因 Next.js SPA 路由不会自动算页面访问）
  - `download_click`（仅在下载按钮处上报）

### 2.2 说明

- **文章详情页当前没有下载按钮**，因此**不会计算文章页转化率**。
- 文章页只用于判断 SEO 流量与内容表现。

## 3. 数据指标是什么（最少集合）

### 指标 A：首页访问量
- **定义**：首页 `page_view` 数量（按语言/国家拆分）
- **用途**：判断首页是否承接到足够流量

### 指标 B：内容页访问量
- **定义**：`/date-ideas` 列表页 + 文章详情页的 `page_view`
- **用途**：衡量 SEO 内容带来的真实流量

### 指标 C：下载点击量
- **定义**：`download_click` 事件数量（App Store / APK 分开统计）
- **用途**：核心转化结果

### 指标 D：下载点击率（首页）
- **定义**：`download_click / 首页 page_view`
- **用途**：衡量首页下载引导是否有效

> 说明：目前下载按钮只出现在首页轮播（MainCarousel），所以下载点击率只对首页成立。

## 4. 这些指标反映在业务上是什么意思

- **首页访问量高 + 下载点击率低**
  - 说明流量有，但转化弱 → 需要优化 CTA 文案、按钮位置、首屏价值表达。

- **内容页访问量高 + 首页访问量低**
  - 说明 SEO 内容有效，但承接不足 → 需要加强内容页导流到首页或增加下载入口。

- **下载点击量高**
  - 说明转化策略生效 → 继续复用当前首页结构与内容模板。

- **某语言下载点击率显著更高**
  - 说明该语言用户更易转化 → 优先扩展该语言 SEO 内容。
